# db-bot 사용 안내

이 문서는 Discord 서버에서 `db-bot`을 사용하는 방법을 정리한 사용자용 안내서다.  
학습 계획 생성은 `db_study_plan` 채널에서 진행하고, 학습 자료 확인과 제출/질문은 각 전용 채널에서 진행한다.

## 시작 전 알아둘 점

- `db_study_plan` 채널 입력은 서버장만 가능하다.
- 학습 기간 입력 형식은 `숫자 + D` 이다.
- 코스는 `입문자`, `중급자`, `상급자` 중 하나를 선택한다.
- 코스 미리보기 확인 단계에서는 `확인` 또는 `취소`만 입력할 수 있다.
- 상세 일정 확인 후에는 `시작` 또는 `취소`만 입력할 수 있다.

## 채널 구조

### `db_study_plan`

학습 계획을 만들고 관리하는 메인 채널이다.

여기서 할 수 있는 일:

- 새 학습 시작
- 학습 기간 입력
- 코스 선택
- 상세 일정 생성
- 학습 시작
- 진행 중 코스 조회
- 중단 / 재개 / 종료

### `db_daily`

하루 학습 자료가 게시되는 카테고리다.

- `db_tutor`
  오늘 학습 설명이 올라온다.
- `db_quiz`
  오늘 문제와 힌트가 올라온다.
- `db_answer`
  모범 답안과 해설이 올라온다.
- `user_answer`
  답안을 제출하는 채널이다.
- `user_ask`
  학습 질문을 하는 채널이다.

## 처음 시작하는 방법

`db_study_plan` 채널에서 아래 순서로 진행한다.

1. `새 학습` 입력
2. 기간 입력
3. 코스 선택
4. 코스 미리보기 확인 후 `확인` 입력
5. 상세 일정 확인 후 `시작` 입력

예시:

```text
새 학습
10D
중급자
확인
시작
```

## `db_study_plan` 주요 명령

- `새 학습`
  새로운 학습 계획을 시작한다.
- `-h`
  전체 명령과 흐름을 본다.
- `리스트`
  현재 학습 중인 코스를 본다.
- `중단리스트`
  현재 중단된 코스를 본다.
- `완료리스트`
  완료한 코스를 본다.
- `N번 코스 중단`
  진행 중 코스를 중단한다.
- `N번 코스 재개`
  중단된 코스를 다시 시작한다.
- `N번 코스 종료`
  중단된 코스를 종료 처리한다.

## 학습 자료 보는 방법

### `db_tutor`

오늘 학습할 내용을 설명하는 채널이다.

- 개념 설명
- 예시
- 실무 관점 설명
- 주의점

### `db_quiz`

오늘 풀 문제와 힌트가 같이 올라오는 채널이다.

- 문제는 번호별로 게시된다.
- 각 문제 아래에 힌트 1~3이 함께 포함된다.
- 힌트만 따로 올라오는 별도 채널은 사용하지 않는다.

### `db_answer`

모범 답안과 해설이 게시되는 채널이다.

- 모범 답안은 스포일러 코드블록 형태로 표시될 수 있다.
- 해설은 스포일러로 표시될 수 있다.

## 답안 제출 방법

답안 제출은 `user_answer`의 해당 스레드에서 진행한다.

형식:

````text
!제출 문제 1
```sql
SELECT *
FROM example_table;
```
````

제출 규칙:

- `!제출 문제 N` 형식이어야 한다.
- 답안은 코드블록 안에 작성해야 한다.
- 각 문제는 최대 3번까지 제출할 수 있다.
- 제출 결과는 DM으로 안내된다.
- 제출 확인 메시지와 사용자 메시지는 일정 시간이 지나면 자동으로 삭제될 수 있다.

## 질문하는 방법

질문은 `user_ask`의 해당 스레드에서 진행한다.

형식:

```text
!질문 서브쿼리와 조인 차이를 오늘 내용 기준으로 다시 설명해주세요.
```

질문 규칙:

- 하루 최대 10번까지 질문할 수 있다.
- 오늘 학습 내용과 관련된 질문만 답변한다.
- 정답을 직접 알려달라는 요청은 제한될 수 있다.
- 기존 질문과 유사하면 답변이 재사용될 수 있다.

## 제한 사항

- 학습 기간 입력 범위: 최소 5일, 최대 100일
- 질문: 하루 최대 10회
- 문제 제출: 문제별 최대 3회
- 동시에 진행 가능한 ACTIVE 코스 수는 제한될 수 있다

## 생성 중 안내가 뜰 때

코스 미리보기, 상세 일정, 1일차 자료 생성 시 생성 중 안내 메시지가 보일 수 있다.

- 보통 수 초에서 90초 정도 걸릴 수 있다.
- 3분 이상 오래 걸리면 `취소` 후 다시 시도한다.
- 같은 코스의 시작 처리가 이미 진행 중이면 잠시 기다렸다가 완료 메시지를 확인한다.

## 자주 있는 상황

### 봇이 입력을 이해하지 못할 때

- `-h`를 입력해 현재 가능한 명령을 다시 확인한다.
- `db_study_plan`에서는 현재 단계에 맞는 입력만 해야 한다.

### 제출 형식 오류가 날 때

아래 형식인지 다시 확인한다.

````text
!제출 문제 1
```sql
SELECT *
FROM example_table;
```
````

### 질문에 답하지 않을 때

아래 경우 답변이 제한될 수 있다.

- 오늘 학습 내용과 관련 없는 질문
- 정답을 직접 요구하는 질문
- 제공된 학습 문맥만으로 안전하게 답변하기 어려운 질문

### 이미 3회 제출했다고 나올 때

같은 문제에 대해 이미 3번 제출한 상태다.

- 다른 문제는 계속 제출할 수 있다.
- 같은 문제는 더 이상 제출할 수 없다.

### 필수 채널이 없다고 나올 때

- 학습 시작에 필요한 채널이 누락된 상태다.
- 채널을 다시 만든 뒤 `중단리스트`에서 재개하면 된다.

## 운영자 메모

- `db_study_plan`은 서버장이 학습 계획을 조작하는 채널이다.
- `db_tutor`, `db_quiz`, `db_answer`, `user_answer`, `user_ask` 채널이 정상적으로 있어야 한다.
- 첫 시작 후 1일차 자료가 생성되면 이후 일차는 일정 날짜의 오전 10시에 자동 게시된다.

## 빠른 요약

- 새 학습 시작: `새 학습`
- 기간 입력: `10D`
- 코스 선택: `입문자`, `중급자`, `상급자`
- 미리보기 확정: `확인`
- 학습 시작: `시작`
- 답안 제출: `!제출 문제 N` + 코드블록
- 질문 입력: `!질문 질문내용`
